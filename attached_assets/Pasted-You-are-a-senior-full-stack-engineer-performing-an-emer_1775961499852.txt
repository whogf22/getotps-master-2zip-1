## ⛔ CRITICAL (6) — Money, security, uptime. Fix these FIRST.

**CRIT-1: trust proxy not set — rate limiting broken for all users behind proxy.**
Add `app.set('trust proxy', 1)` BEFORE all middleware in the main Express setup file. Without this, `req.ip` returns the proxy's IP and rate limiting treats every user as one person.

**CRIT-2: Session uses default MemoryStore — memory leak, will crash in production.**
Install `connect-pg-simple`. Replace the default MemoryStore with a PostgreSQL-backed session store:
```ts
import pgSession from 'connect-pg-simple';
const PgStore = pgSession(session);
app.use(session({
  store: new PgStore({ conString: process.env.DATABASE_URL, tableName: 'user_sessions', createTableIfMissing: true }),
  secret: process.env.SESSION_SECRET,
  resave: false, saveUninitialized: false,
  cookie: { secure: process.env.NODE_ENV === 'production', httpOnly: true, maxAge: 86400000, sameSite: 'lax' }
}));
```
Generate a random 64-char SESSION_SECRET if missing.

**CRIT-3: No expired order cleanup — user funds stuck forever in "pending" orders.**
Create a cleanup function that runs every 5 minutes + once on startup. It must: find all orders with status `pending` older than 15 minutes, then inside a `db.transaction()`: refund `order.cost` to `users.balance` and set order status to `expired`. Add `expired` to the status type if missing. Log every refund.

**CRIT-4: No foreign key constraints — orphaned records possible.**
Add `.references(() => table.id, { onDelete: 'cascade' })` to EVERY relational column across ALL Drizzle schema tables (orders, transactions, rentals, api_keys, audit_logs, etc.). Use `onDelete: 'cascade'` for child records, `onDelete: 'set null'` where appropriate. Run `npx drizzle-kit generate` then `npx drizzle-kit migrate`.

**CRIT-5: Cancel/refund not transactional — server crash = user loses money.**
Find ALL cancel/refund endpoints. Wrap every one in `db.transaction()` with row-level locking (`.for('update')`). The balance update AND status change MUST happen in a single transaction. This applies to order cancellations, rental cancellations, and any other money-moving operations.

**CRIT-6: API response logging leaks sensitive data (API keys exposed in logs).**
Create a `sanitizeForLogging()` utility that recursively redacts keys matching: apiKey, api_key, token, password, secret, authorization, cookie, sessionId, balance. Run `grep -rn "console.log" server/` and apply sanitization to every log statement that outputs request/response data.
